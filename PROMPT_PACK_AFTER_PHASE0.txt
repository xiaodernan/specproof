## P0.5：补完真实性

先把第一周欠下的核心闭环补齐：

### 必须完成项
1. Maven Wrapper 实际执行（JUnit 在真实 Base/Head 环境运行）
2. 真实 Base/Head Spring Boot 差分
3. HTTP 和 MySQL 状态差分
4. 至少一个真实 LLM 生成 JUnit（DeepSeek 生成 → 结构校验 → 编译 → 自动修复编译错误 → Base/Head 分别执行 → 生成 Evidence）
5. 干净容器重放 Capsule
6. Holdout、负样本和对抗样本
7. 与"直接让 DeepSeek 看 Diff"的基线比较

### DeepSeek 注意事项
- 第三方网关，思考模式结合 Tool Calls 时不能照搬普通 OpenAI 调用
- DeepSeek 官方要求涉及工具调用的后续请求完整回传 reasoning_content
- Provider Adapter 必须专门处理或降级

### 验收标准
- 一条命令完成完整 Base/Head 测试执行
- 至少一个 DeepSeek 生成的 JUnit 测试成功编译并运行
- Capsule 在干净容器中可重放
- 与 DeepSeek Diff Review 基线的对比数据

### 禁止
- 没有 Maven/JUnit 实际执行就声称"验证通过"
- 用静态规则发现替代可执行证明
- 在此步通过前开始堆后续功能

### LangGraph 可靠性

### 可观测性
- OpenTelemetry Trace
- Context Propagation：跨 HTTP、进程和 MQ 传播上下文

### 注意
- RabbitMQ 的 Publisher Confirm 和 Consumer Ack 是两个不同方向的可靠性机制，不能只做其中一个
